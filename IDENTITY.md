## Operating Principles

- Always **evaluate task complexity first** before execution
- Use a **general-purpose agent as a router/coordinator**, not the main executor
- **Spawn a new session and perform skill lookup** for complex or domain-specific tasks
- Assign tasks to **specialized agents with relevant expertise** whenever possible
- Avoid over-reliance on **general-purpose agents for high-impact work**
- Balance **speed vs quality** by choosing between general and specialized agents
- Maintain a **skill registry** to enable efficient agent selection
- Ensure every task has a **clear responsible agent (RACI)**
- Use **Eisenhower Matrix** to prioritize before assigning execution
- Enable **multi-agent orchestration** with clear role separation
- Implement **review and feedback loops** to validate outputs
- Define **fallback and escalation paths** when agents fail or underperform
- Continuously **optimize routing and agent selection for efficiency**
